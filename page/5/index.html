Git Practice - Getting Started

发表于 2017-10-23 更新于 2019-05-21 分类于 VCS
本文字数： 1.3k 阅读时长 ≈ 1 分钟

Start a working area

init

  • 在当前目录[指定目录]初始化工作空间

clone

  • 将指定工作空间拷贝到当前目录[指定目录]

Work on the current change

add

  • 将文件添加到中转区（路径/文件名）

  • 将文件添加到中转区（文件通配符，匹配到的文件都将会添加到中转区）

reset

  • 将HEAD重置到某一个状态（路径/文件名，若不指定则对所有提交生效）

rm

  • 删除指定文件（文件名/文件通配符）

Examine the history and state

status

  • 查看当前仓库状态

log

  • 查看仓库变更日志

Grow, mark and tweak your common history

commit

  • 将中转区的变更提交到仓库（说明内容）

checkout

  • 切换分支（–路径/文件名）

  • 切换分支（<分支名称>）

branch

  • 创建分支（<分支名称>）

  • 删除分支（<分支名称>）

diff

  • 查看更改差异：查看上次提交的不同之处

  • 查看更改差异：HEAD为最新提交指针代码

  • 查看更改差异：查看中转区的更改差异

merge

  • 合并其他分支到当前分支（<其他分支名称>）

Collaborate

push

  • 将本地分支推送到远程仓库（<远程仓库名称> <本地分支名称>）

-u告诉Git记住参数，这样下次我们可以简单地运行git push

pull

  • 拉取远程仓库变更到本地分支（<远程仓库名称> <本地分支名称>）

remote

remote add

  • 添加远程仓库（<远程仓库名称> <远程仓库地址>）
